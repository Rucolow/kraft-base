# KRAFT BASE スタッフアプリ 改善計画（提案）

Fable 5 による全機能レビュー（10観点 × 84指摘）を統合したもの。**本番には一切未反映＝提案のみ**。
上位項目は実コードで裏取り済み。中〜低の一部はレビュー報告ベース（会話上限で自動検証フェーズが中断したため、着手前に実機/コードで最終確認する）。

前提の現状:
- **本番反映済み**: ゲスト編集のデータ消失／キャンセル数／タスク削除／チェックイン各種（複数名・再入力・二重送信・キオスク長押し）／04:00境界／per_checkout／勤務集計／ログイン再接続 等。
- **ブランチ待機（ステージング検証待ち）**: コネクタ型変換(serialize.ts)、`0015_staff_claim_guard`、`0016_device_org_member`、境界テスト。→ 本計画の「同期のズレ」「権限昇格」「device同期」はここに対応済み。

優先度: **P0=業務停止/法令/情報漏えいリスク** / **P1=日々の使い勝手・性能・アクセシビリティ** / **P2=保守性・運用・仕様残**。
影響先: [FE]=フロントのみ・AI検証可 / [DB]=本番DB/RLS要 / [OPS]=運用/インフラ / [STG]=ステージング検証必須。

## P0 — まず直すべき（信頼性・法令・情報保護）

### P0-1. オフラインで“使えるはず”が使えない三点セット [FE]
ローカルファーストの中核公約が、実際には次で破れている:
- **古いビルドが居座る (SW更新なし)** — `vite.config.ts` は `registerType:'autoUpdate'` のみで、`virtual:pwa-register` の利用がコード上に一切ない（周期`update()`もリロード導線も無し）。受付iPadは開きっぱなしなので**新ビルドが反映されない**（オーナーが実際に踏んだ“clear site data”事件）。→ `useRegisterSW`で1時間ごと＋復帰時に`update()`、`controllerchange`で**/checkin中は避けて**安全にリロード、Setup画面にビルド版数表示。**(M, 効果大)**
- **オフライン起動でログイン画面に締め出し** — `auth.tsx` は起動時 `getSession()` 一発。期限切れトークン(通常1h)＋回線不通だと `session=null`→`RequireApp`が`/login`へ飛ばす＝**手元のローカルデータ（本日ゲスト・緊急連絡先）に入れない**。→ 「保存済みセッションあり＋ネットワーク失敗」はオフライン認証済みとして**ローカルに通す**、`online`復帰で再同期。**(M)**
- **起動直後の白画面／初期化失敗で永久白画面** — `PowerSyncProvider` は `startPowerSync()`(＝`db.connect`待ち)が解決するまで`null`。遅い回線で毎回白画面、`db.init`失敗時は`.catch`が無く**恒久白画面**。→ `db.init`だけ待ち`connect`は非同期化、スプラッシュ＋エラー/再試行表示。**(S)**

### P0-2. 同期が壊れても気づけない [FE＋一部STG]
`SyncBadge` にエラー状態・最終同期時刻が無く、コネクタは失敗アップロードを**黙って破棄**するので画面は「同期済み」に見える。→ SyncBadgeに「未同期n件／同期失敗・再試行／最終同期HH:MM」を出す。※根っこの「配列/boolが同期で消える」は**ブランチのコネクタ型変換で対応済み（要ステージング実証）**。**(S–M)**

### P0-3. 法定宿泊者名簿(passport)のライフサイクル [DB＋FE＋OPS]
- **閲覧・書き出し・バックアップ導線が皆無** — 入るだけで出せない。3年保管義務があるのに。→ オーナー用の**名簿一覧＋CSV/PDFエクスポート**。
- **ゲスト削除で名簿も消える** — `checkin_record.guest_id ... on delete cascade`＋名簿自身に宿泊日が無い。→ カスケード廃止（または論理削除）、`stay_date`列と“記入バッチ”識別子を追加（再入力の重複行問題 M26 も解消）。
- **全端末に全名簿を無期限複製** — グローバル同期バケットで、各iPadのOPFSに全passportが溜まり続ける。→ 同期の**日付ウィンドウ化**＋古い名簿のサーバ保持/端末側パージ方針。**(合計 L、法令に関わるので優先)**

### P0-4. 端末紛失・キオスク越え・写真公開 [FE＋DB]
- **サインアウトでローカルDBを消さない** (`auth.tsx` は `supabase.auth.signOut()` のみ)＝紛失iPadにpassportが残る。→ サインアウトで `db.disconnectAndClear()`、必要なら OPFS 暗号化キー。
- **/checkin からブラウザ/PWAの戻るでスタッフ全画面へ** — 長押しガードはアプリ内ボタンだけ。ゲストに渡す前提なのに戻る操作で全ゲストPIIに到達可。→ チェックインを独立コンテキスト化（履歴を残さない/戻る抑止/完了後ロック）。
- **写真が公開バケット** — `photoSrc` は `getPublicUrl`＝URLを知れば誰でも閲覧、かつバケット未作成。→ **非公開バケット＋署名URL**、バケットのプロビジョニング。**(S–M)**

## P1 — 日々の使い勝手・性能・アクセシビリティ

### P1-1. 性能・容量（古いiPad×宿wifi） [FE]
- **プリキャッシュ 8.3MB→約2MB** — wa-sqlite の4変種中3つ(約6.3MB)は実行時に読まれない。`globIgnores`で除外＋`maximumFileSizeToCacheInBytes`削除。**(S・効果大)**
- **フォントが外部Google Fontsで描画ブロッキング＆SW未キャッシュ** — オフライン/低速wifiで初回描画が大幅遅延（本環境で実測13s、良回線なら速い）。→ **フォント自己ホスト＋SWプリキャッシュ**。**(S–M)**
- **写真が本番では無圧縮フルサイズ**（圧縮はdemo経路のみ）。→ 本番でも圧縮。**(S)**

### P1-2. アクセシビリティ（非IT・高齢・外国人） [FE]
- **主要ボタン文字のコントラスト不足**（オレンジ地にクリーム文字 = 1.97:1 ダーク / 2.31:1 ライト、WCAG不合格）＋**ライトのオレンジ/wood系テキスト 2.3–3.1:1**。→ ボタンは濃色文字、テキスト用オレンジは `orange-deep` 系へ。**(S–M)**
- **フォーカス可視化なし**（全入力 `outline-none`）、**タップ標的<44px**（編集鉛筆16px・無ラベル）、**11–12px文字の下限**と端末別文字サイズ調整。**(S–M)**
- **キオスクのチェックインが日英のみ** — 熊野古道は独/仏/伊/西の巡礼者が多い。ゲストの言語は既知なので**その言語で表示**。**(M)**
- **スタッフUIが日本語のみ** — 英語のみのヘルパーが操作するかを明示決定（するなら i18n）。**(方針決め)**

### P1-3. 業務フローの手当て [FE中心]
- **過去ゲストを探せない**（一覧は今日/これから先のみ）→ 履歴タブ/名前検索。
- **朝のOTA入力**：連続追加(save & add next)・日付保持・連泊コピーが無い。
- **ベッドの重複割当が見えない** → 割当時に警告。
- **申し送り(followup)の完了が追跡不能＆ゲスト紐付けUIなし**、`task.owner_id`/`@担当`が常にnull。
- **明示的な「シフト終了」が無い**＝勤務が翌04:00まで水増し。
- **深夜の引き継ぎダイジェストが前夜のタイムラインを取りこぼす** (`deriveDigest` の since 起点)。
- 細かいUX: 忘れ物/商品ステータスが“静的バッジに見えるボタン”で5状態循環／商品削除が価格入力の隣で無確認／ShiftGateの二重タップ／スタッフ交代時の開始レース／コックピット「全員到着済み」誤表示／レビュー依頼一覧にキャンセル客が残る。**(各S、まとめて中)**

## P2 — 保守性・運用・仕様残

### P2-1. テスト＆エラーハンドリング [FE]
- **E2Eがリポジトリ外(/tmp)** — 各フェーズの検証公約を満たすため、Playwright一式を**リポジトリ化しCIで実行**（demoモード＝vite preview）。testing-library/jsdomは導入済みだが未使用。
- **全書き込みが投げっぱなし**（await のみ・try/catch無し）＝失敗が無言、pending旗が固まる（ShiftGate/ContentReader）。→ 共通の書き込みラッパ＋失敗トースト。
- 内容編集の同時編集で後勝ち上書き、アップロードの部分適用。

### P2-2. 運用・インフラ [OPS]
- **Postgresのバックアップ/復元手順が無い**（法定名簿なのに）→ Supabase PITR等の方針。
- **監視・アラート無し** — 無料枠の**一時停止(オフシーズンのアイドル)**に気づけない。
- **staging schema.sql が既にドリフト**（0001-0014記載・実体0016）→ 再生成スクリプト。
- **1ページ運用ランブック** (`docs/ops-runbook.md`) を作成：デプロイ確認・同期異常時・端末紛失時・名簿エクスポート手順。

### P2-3. 仕様残（build-plan） [FE]
- **Phase6 Web Push 未実装**（3つの通知トリガーがアプリ外に出ない）。
- **文脈手順レイヤ未配線**（`useProcedureForPhase` がデッドコード、GuestDetail/Todayに手順が出ない）。
- 割当/リンク系フィールド(owner_id/guest_id)のUI。

## 補正メモ（過大/重複の整理）
- 「見知らぬ他人が全名簿を同期(H17)」は **`guest_select = is_org_member` により実際には不可**（メンバーでないとSELECTできない）。残る実リスクは**権限昇格＝`0015`で対応済み**＋Supabaseのサインアップ開放可否の締め（allowlist）。
- 「権限昇格(M37)」「device同期(M38/0016)」「配列同期(コネクタ)」は**ブランチで対応済み**、ステージング実証待ち。
